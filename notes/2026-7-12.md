## 重要知识点

### 1. `printf` 是有返回值的
- `printf` 返回的是**成功打印到屏幕上的字符个数**。
- 利用 `int digits = printf("%d", n);` 可直接得到数字 `n` 的十进制位数。
- **坑点**：负数会多算一位（负号也算字符）。

### 2. `fgets` —— 读取含空格的字符串
- **为什么不用 `scanf("%s")`？** `scanf` 遇到空格就停止，无法读取 "Hello World" 这类输入。
- **函数原型**：`char *fgets(char *str, int size, FILE *stream);`
- **三个参数**：
  1. `str`：存放输入字符数组（“篮子”）。
  2. `size`：篮子大小（最多读 `size-1` 个字符，留一个给 `\0`）。
  3. `stream`：数据源，从键盘读就填 `stdin`。
- **核心规则**：
  - 读到换行符 `\n` 或装满 `size-1` 个字符或遇到 `EOF` 时停止。
  - 它会把键盘敲的**回车键（`\n`）**也当作普通字符存进数组！

### 3. `strcspn` —— 去除 `fgets` 带回的换行符
- 函数原型：`size_t strcspn(const char *str, const char *reject);`
- 作用：在 `str` 中查找 `reject` 里任意字符第一次出现的位置。
- 用法：`str[strcspn(str, "\n")] = '\0';`
  - 先计算出换行符 `\n` 在数组中的下标位置；
  - 再用 `\0`（字符串结束符）替换掉这个位置的字符。
- **边界情况**：如果 `fgets` 因为数组满了没读到 `\n`，`strcspn` 返回字符串长度，此时 `str[长度] = '\0'` 相当于给已有结束符再赋一次，等于没操作，完美兼容。

## 今日遇到的报错与调试

### 报错 1：`fatal error: xxx.c: No such file or directory`
- **原因**：终端当前工作目录（`pwd`）不在 `.c` 文件所在的文件夹。
- **解决**：使用 `cd /mnt/d/CS-Learning/c-practice` 切换到正确目录后再编译。

### 报错 2：`undefined reference to 'main'`
- **原因**：`.c` 文件中没有写 `main` 函数，或者把 `main` 拼写错误（如写成了 `mian`）。
- **解决**：检查代码是否包含 `int main() { ... }` 完整结构。
